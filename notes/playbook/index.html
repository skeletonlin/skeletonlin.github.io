Engineering Playbook (WIP)
A reusable checklist for debugging, profiling, deployment, and release. This page reflects my approach to systematizing engineering work.
可複用的檢查表：debug / profiling / deployment / release（這頁是統整能力的證據）。
1) When performance drops
  • Confirm baseline: input resolution, model version, config version.
  • Observe CPU usage & identify saturated stage (e.g., post-process).
  • Measure steady-state FPS; plan to add latency (mean/p95).
  • Change one variable at a time; keep notes of before/after.
2) Deployment checklist
  • Class mapping: confirm class count + label order.
  • Memory budget: pre-allocate buffers, reduce copies.
  • Sanity test: visual output check vs baseline.
  • Stability test: run sustained duration; monitor memory growth.
你提過的兩大坑（class 不對、記憶體不足）已經被整理成 checklist。
3) Regression strategy
  • Visual regression: overlay outputs side-by-side.
  • Functional regression: key scenarios must still trigger expected events.
  • Future: add numeric tolerance checks for intermediate tensors.
4) Release readiness (WIP)
  • Record versions: firmware / model / config hashes.
  • Document known limitations and safe fallback behavior.
  • Prepare quick rollback plan.
