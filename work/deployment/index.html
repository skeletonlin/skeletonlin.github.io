Deployment Workflow (YOLO → ONNX → NPU)
A reproducible workflow for bringing models into embedded deployment with clear configuration alignment.
可重現的部署流程：把模型帶到嵌入式端，並確保設定一致與可維護。
Workflow Overview
  • Export model → ONNX
  • Validate ONNX I/O shapes & output semantics
  • Integrate into NPU toolchain / SDK
  • Align post-processing parameters (NMS / thresholds / label map)
  • Run end-to-end sanity test (visual + basic metrics)
Common Pitfalls & Checklist
1) Class mismatch
  • Confirm model output channels match expected class count.
  • Verify label ordering & mapping (class id ↔ label).
  • Align SDK configuration with the exported model (anchors / strides / decode parameters if applicable).
class 數不對：確認輸出維度、label mapping、SDK 參數（decode/NMS）一致。
2) Memory limitation
  • Pre-allocate buffers and avoid repeated malloc/free in runtime.
  • Reduce intermediate copies (prefer zero-copy / shared buffers when possible).
  • Adjust input resolution if memory budget is tight.
記憶體不足：預先配置 buffer、減少 copy、必要時調整解析度。
Validation (Minimum)
  • Visual sanity check: boxes / labels / confidence look reasonable.
  • Stability: run for sustained duration and monitor CPU/memory.
  • Regression: compare against a baseline build when available.
