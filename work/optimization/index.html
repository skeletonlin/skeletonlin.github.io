Case Study: YOLO NMS Acceleration (+63%)
Problem. In a real-time AI camera pipeline, CPU usage hit ~100% during post-processing (NMS), becoming the throughput bottleneck.
問題：在 AI 攝影機即時 pipeline 中，後處理（NMS）導致 CPU 逼近 100%，成為效能瓶頸。
Hypothesis
  • Vectorize compute-heavy parts of NMS using ARM NEON SIMD to reduce CPU cycles.
  • Improve memory access patterns to reduce cache misses and redundant loads.
假設：用 ARM NEON 向量化 NMS 熱點計算、並調整記憶體存取以降低 CPU cycles。
Approach
  • Identify hotspots by observing sustained CPU saturation during inference runtime.
  • Rewrite key loops with NEON intrinsics and reduce unnecessary memory copies.
  • Keep output behavior consistent with baseline implementation.
作法：定位熱點 → NEON intrinsics 重寫核心迴圈 → 減少 copy → 確保輸出行為與 baseline 一致。
Validation
  • Primary metric: steady-state FPS improvement under fixed input set.
  • Regression check: side-by-side visual comparison of overlay output against baseline.
  • Result: post-processing speed improved by +63% (YOLO NMS).
驗證：固定輸入集測 FPS，並以可視化輸出比對做回歸檢查；YOLO NMS 提升 +63%。
What I would improve next
  • Add end-to-end latency measurement (mean / p95) and per-stage timing breakdown.
  • Add numeric tolerance checks for intermediate tensors to complement visual regression.
下一步：補上 end-to-end latency（mean/p95）與分段 profiling；加入數值誤差檢查強化驗證。
