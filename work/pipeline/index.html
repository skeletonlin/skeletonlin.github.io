End-to-end Embedded AI Pipeline
A layered design: Input → Processing → Decision → Actuation
分層架構：輸入 → 處理 → 決策 → 執行（警示/控制）。
1) Dataflow (High-level)
  • Input: Camera capture & frame buffer management
  • Processing: ISP tuning → NPU inference → post-processing
  • Decision: event aggregation / state machine / debouncing
  • Actuation: OSD overlay + MCU IO alerts (buzzer / LED)
2) Interfaces (What connects components)
  • Frame contract: resolution / pixel format / stride / buffer ownership
  • Model contract: input tensor shape, output tensor semantics
  • Event contract: standardized event fields (type, confidence, timestamp)
  • Actuation contract: alert commands (on/off, duration, rate limit)
介面重點：frame/模型輸出/event/警示控制的「契約」要能被跨模組維護。
3) Latency & Reliability (Engineering constraints)
  • Latency budget: keep pipeline stable under sustained load (avoid CPU saturation)
  • Memory: pre-allocation & reduced copies (avoid fragmentation)
  • Fail-safe: define behavior when inference becomes invalid / no signal
  • Stability test: sustained runtime monitoring (CPU / memory growth)
4) Decision & Validation (How I work)
  • Observation: identify bottlenecks (e.g., CPU saturation in post-processing)
  • Proposal: suggest improvements (NEON, memory layout, thresholds)
  • Alignment: align approach with lead before integration
  • Verification: baseline comparison + visual regression + FPS stability
